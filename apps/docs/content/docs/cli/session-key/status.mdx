Checks if a session key is installed for a smart account on a given chain. You can optionally provide a custom RPC URL.

## Usage

```bash
namera session-key status [--alias <name>] [--chain <chain>] [--rpc-url <url>]
```

Aliases:

- Group alias: `namera sk`
- Command alias: none

## Flags

- `--alias`, `-a`: Session key alias
- `--chain`: Chain name (must be a supported chain key)
- `--rpc-url`: Custom RPC URL for the selected chain

Global flags:

- `--params`: Raw JSON parameters for agent-driven execution
- `--output`, `-o`: `pretty`, `json`, or `ndjson`
- `--quite`, `-q`: Suppress output

If `--alias` or `--chain` is missing, the CLI will prompt you to select them. `--rpc-url` is optional.

## Params mode (agents)

Use `--params` with the JSON schema for the command:

```bash
namera schema session-key.status
```

Expected shape:

```json
{
  "alias": "my-session-key",
  "chain": "eth-mainnet",
  "rpcUrl": "https://rpc.ankr.com/eth"
}
```

`rpcUrl` is optional. If omitted, the CLI uses its default public RPC.

## Output

On success, the command returns:

- `installed`: `true` or `false`

## Examples

Explicit chain:

```bash
namera session-key status -a my-session-key --chain eth-mainnet
```

Custom RPC:

```bash
namera session-key status --alias my-session-key --chain eth-mainnet --rpc-url https://rpc.ankr.com/eth
```

Params mode (agents):

```bash
namera session-key status --params '{"alias":"my-session-key","chain":"eth-mainnet"}'
```

Short alias group:

```bash
namera sk status -a my-session-key --chain eth-mainnet
```

Schema lookup:

```bash
namera schema session-key.status
```
